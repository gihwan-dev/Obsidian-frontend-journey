## 📝 Daily Diary

### 🖥️ 개발 활동

#### Obsidian Vault TODO 관리 설계
- 프로젝트별 TODO를 볼트 안에서 일관되게 다루기 위해 `docs/design/todo-management-automation.md`를 새로 정리했고, 루트 보드는 읽기 전용 허브로 두는 방향이 문서로 남았다.
- 활성 프로젝트의 기준을 `프로젝트 폴더 + 직계 TODO.md`로 좁히고 `Backlog -> Todo -> Doing -> Done` 구조와 체크리스트 카드 형식을 고정해, 이후 정리와 자동화가 같은 기준을 따를 수 있게 했다.
- 작업 이동을 같은 프로젝트 안의 상태 변경으로 한정하고 브리핑 범위를 모든 활성 프로젝트로 보는 쪽으로 수렴시켜, 다음 결정이 시간 단위 TODO 브리핑 계약 확정으로 이어지게 했다.

### 📓 기록 활동
- 새로 작성한 노트: [[docs/design/todo-management-automation]]

### 🧾 커리어 자산
- 분산된 Obsidian TODO 체계를 프로젝트별 단일 보드 구조로 정리하는 설계 문서를 만들고, 보드 스키마와 자동화 경계를 명확히 정의했다.

### 💭 회고
오늘은 자동화를 바로 만들기보다, 나중에 흔들리지 않을 운영 규칙을 먼저 고정하는 데 집중했다.
이제 남은 핵심은 시간 단위 브리핑이 어떤 입력을 읽고 어떤 형태로 남아야 하는지까지 마저 닫는 일이다.
