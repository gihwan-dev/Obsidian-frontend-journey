## 📝 Daily Diary

### 🖥️ 개발 활동

#### claude-setup
- Codex 기반 작업 흐름을 더 가볍고 예측 가능하게 유지하려고 `plan/build` 체계를 다시 다듬고 fallback 없는 실행 원칙과 병렬 작업 독립성을 정리했고, 이후 스킬들이 같은 운영 계약을 따를 기반이 남았다.
- 검증 루프와 프롬프트 템플릿을 손봐 lint·format 확인을 기본화하고 반복 가능한 리뷰 루프를 정리했고, 실제 작업에서 더 일관된 품질 게이트를 적용할 수 있게 됐다.

#### gitlab-slash-command
- 이슈 #33을 장기 실행에 더 맞는 구조로 옮기기 위해 LangGraph HIL durable execution 아키텍처로 전환했고, 이후 slash command 작업이 상태를 보존한 채 이어질 기반이 남았다.
- triage와 handoff 단계에서 git context를 더 보수적으로 반영하도록 accept 규칙과 git log 도구를 보강했고, 작업 인수인계 판단을 코드 변경 근거에 더 가깝게 맞출 수 있게 됐다.

#### dev-report-hub
- MR 결과를 작업 흐름 안에서 바로 공유하려고 자동 댓글 기능을 설계에서 구현까지 연결했고, 리포트 저장 이후 업로드와 후속 처리를 붙일 수 있는 확장 방향이 남았다.
- 새 기능이 배포 단계에서 막히지 않도록 CLI 빌드 경로와 의존성 누락을 함께 정리했고, 댓글 기능을 실제 운영 환경에 붙일 준비가 됐다.

#### k8s-manifests
- gitlab-slash-command의 durable execution을 실제로 받쳐주기 위해 클러스터 내부 PostgreSQL 배포와 차트 구성을 추가했고, 애플리케이션 전환에 필요한 저장소 기반이 남았다.
- 체크포인터와 의존성 대기 설정까지 묶어 배포 순서를 안정화했고, 서비스가 DB 준비 이전에 올라오며 깨지는 위험을 줄였다.
