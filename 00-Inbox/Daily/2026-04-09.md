## 📝 Daily Diary

### 🖥️ 개발 활동

#### gitlab-slash-command
- 멘션과 슬래시 커맨드에도 워커 기반 병렬 처리를 확장하려고 설계와 구현 방향을 구체화했고, 이슈 자동화와 별도로 어떤 경로에 병렬성이 빠져 있는지 정리해 다음 작업 기준을 남겼다.
- reconcile 흐름과 의도 분류를 더 촘촘하게 다듬으려는 맥락에서 파이프라인 실패 원인과 운영 제약을 다시 점검했고, 파이썬 의존성과 웹 리서치 확장 필요성을 후속 이슈로 연결해 두었다.
- 남은 작업은 이슈 자동화까지 같은 병렬 실행 모델로 묶고, 검증 환경 제약 없이 품질 게이트를 재현할 수 있게 실행 경로를 정리하는 일이다.

#### exem-table
- 테이블 셀 렌더링 구조와 이름이 역할을 충분히 설명하지 못하는 문제를 줄이려고 컴포넌트 책임과 네이밍을 여러 차례 재검토했고, 주석과 문서를 보강해 팀이 같은 기준으로 읽을 수 있는 방향을 잡았다.
- 페이지네이션, 다크 테마, 스크롤바, 드래그 앤 드롭처럼 사용자가 바로 체감하는 UI를 피그마와 스토리북 기준으로 계속 대조하면서 어긋난 부분을 수정했고, 확인이 필요한 항목은 GitLab 이슈로 분리해 흐름을 유지했다.
- 남은 결과는 전반적인 z-index와 상호작용 계층을 한 번 더 정리하고, 스토리북이 디자인 검증 도구 역할까지 안정적으로 하도록 세부 케이스를 더 다듬는 것이다.

#### React-Dashboard
- 구버전 Edge 호환성 이슈를 해결하려고 Jenkinsfile 추가와 정적 서빙 전환까지 포함해 배포 경로를 정리했고, 실제 화면에서 깨지는 위치 문제를 재현하며 어떤 브라우저 범위까지 지원할지 판단 근거를 모았다.
- 알람 UI와 모달 정렬 문제를 원인 중심으로 추적하면서 `translate`와 구형 CSS 대응의 충돌을 검토했고, 무리하게 하위 버전을 끌어안기보다 최소 지원 버전을 다시 정하는 쪽으로 방향을 좁혔다.
- 남은 작업은 호환성 기준을 문서와 파이프라인에 명확히 고정하고, 최신 브라우저 기준 레이아웃이 흔들리지 않도록 검증 절차를 붙이는 일이다.
