## 📝 Daily Diary

### 🖥️ 개발 활동

#### claude-setup
- HyperAgent 자기개선 시스템을 세션 분석과 자동 적용까지 이어지는 파이프라인으로 설계하고 구현해, 개인 생산성 도구가 하루의 사용 경험에서 개선점을 끌어올릴 수 있는 기반을 남겼다.
- `design-docs`에 spike 단계를 추가해 런타임 가정을 본격 설계 전에 검증하도록 보강했고, 설계가 실제 실행 조건과 어긋나는 문제를 줄이는 방향으로 다듬었다.

#### gitlab-slash-command
- GitLab mention 워크플로우를 고정된 specialist/팀 라우팅에서 오케스트레이터가 필요한 에이전트를 자율적으로 고르는 구조로 바꿔, 프롬프트가 작업 절차를 과하게 강제하지 않도록 정리했다.
- mention 프롬프트를 base와 overlay 구조로 재구성하고 레거시 orchestrator를 제거해, 페르소나와 작업 맥락은 남기되 중복 지시와 불필요한 도구 설명은 줄였다.
- 자동 커밋과 저장소 도구 경로를 fail-closed 방향으로 보강하고 미사용 코드 탐지 설정을 추가해, 멀티에이전트 흐름의 안정성과 유지보수성을 함께 점검했다.

#### exem-table
- React Table에 Copy Text와 Copy Image 클립보드 기능을 추가하고 브라우저 테스트와 Storybook 사례를 보강해, 사용자가 테이블 데이터를 텍스트와 이미지로 가져갈 수 있는 흐름을 검증 가능한 형태로 남겼다.
- Copy Image 동작 불량과 테스트 격리 문제를 반복적으로 확인하며 `setTimeout`, `requestAnimationFrame`, Radix Portal 상태를 조정했고, 무거운 캡처 작업에는 로딩 오버레이와 에러 처리를 붙여 사용 중 피드백을 보강했다.
- OptionsMenu를 플러그인/레지스트리 구조에서 직접 합성 구조로 다듬고 폴더와 타입 경계를 정리해, 기능 추가와 제거 시 변경 범위가 더 작아지도록 구조를 정리했다.
- 셀 타입 시스템과 헤더 디자인을 보강해 13종 셀 포맷 렌더링과 Figma 기준 배경색을 맞췄고, 테이블 컴포넌트의 표현 범위를 넓히는 작업을 이어갔다.
