## 📝 Daily Diary

### 🖥️ 개발 활동

#### claude-setup
- HyperAgent 자기개선 루프가 임의의 대상을 고치지 않도록 사용자 호출 이력과 세션 분석을 중심으로 다시 좁혔고, 잘못 적용된 개선은 되돌린 뒤 스킬 기반 루프로 정리했다.
- 부정 언어 감지와 gap 분석을 보강해 개선 대상과 판단 기준을 더 선명하게 만들었고, Claude Desktop에서 실행되는 조건에 맞춰 불필요한 SDK 전제를 걷어냈다.
- 흩어진 설계·문서·빌드 계열 스킬을 `architect` 흐름으로 통합하는 방향까지 다듬어, 개인용 에이전트 도구의 유지보수 기준을 다시 맞췄다.

#### gitlab-slash-command
- GitLab mention 워크플로우에서 ClickUp 링크 탐지와 검색 에이전트 흐름을 보강했고, E2E 테스트로 드러난 스키마와 응답 파싱 문제를 수정해 실제 멘션 처리의 안정성을 높였다.
- 멀티에이전트 구조를 정보 수집 중심으로 다시 조정하고 실행 계획과 preflight 분류를 붙여, 서브에이전트가 과하게 작업을 가져가는 문제를 줄이는 방향으로 정리했다.
- Harbor 주소와 Docker vendor 복사 순서, CI 포맷·미사용 export 문제를 고치며 배포와 파이프라인이 막히던 지점을 하나씩 풀었다.

#### exem-table
- React Table의 Copy Image 흐름에서 빈 이미지와 투명 캡처 문제를 다시 추적했고, 가상화 테이블 clone의 너비와 뷰포트 캡처 방식을 다듬어 Storybook에서 확인 가능한 상태로 보강했다.
- 셀 variant 위치와 셀 포맷 API 구조를 정리해 테이블 표현 확장 작업의 변경 경계를 줄였고, 셀 타입 시스템을 이어서 다듬을 기반을 남겼다.
- 실패한 MR 파이프라인과 오래 남은 워크트리까지 확인하며, 기능 수정 뒤 검증 환경을 다시 정리하는 흐름을 유지했다.
