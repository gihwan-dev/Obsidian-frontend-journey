## 목적  
- 운영자가 인스턴스별 경보 이력을 한 화면에서 확인하고 즉시 후속 조치를 수행할 수 있게 한다.  
- 최근 10분의 경보 심각도와 경보 유형 분포를 빠르게 파악할 수 있게 한다.  
  
## 핵심 동작  
- 사용자는 `Graph`와 `Grid` 보기 모드를 전환해 같은 경보 데이터를 요약 화면과 목록 화면으로 확인할 수 있다.  
- 그래프 모드에서는 인스턴스별 5개 경보 영역의 현재 비정상 여부를 원형 상태로 확인할 수 있다.  
- 그래프 모드에서 경보 원형에 마우스를 올리면 해당 인스턴스와 영역의 활성 경보 상세 목록이 표시된다.  
- 그래프 모드에서 경보 원형을 더블클릭하면 해당 인스턴스 기준 1일 경보 이력 화면으로 이동할 수 있다.  
- 그리드 모드에서는 시간 역순의 최근 경보 목록을 확인하고 항목별 상세 동작을 실행할 수 있다.  
- 사용자는 상단 `1-Day History` 액션으로 인스턴스를 선택해 하루 범위 경보 이력 화면으로 이동할 수 있다.  
- 사용자는 우클릭 메뉴로 경보 해제, 목록 비우기, 이벤트 설명 조회, 메모 등록 등 후속 작업을 수행할 수 있다.  
- 사용자는 메모 아이콘으로 현재 목록의 경보들을 메모 등록 화면에 전달할 수 있다.  
- 사용자는 알람 일시정지 아이콘으로 일시정지 목록을 조회하고 개별 항목을 해제할 수 있다.  
  
## 상세 규칙  
- 기본 진입 모드는 그래프 모드이며 플로팅 창으로 열릴 때는 그리드 모드로 시작한다.  
- 그래프 상단에는 최근 10분 누적 `Warning` 수와 `Critical` 수를 함께 표시한다.  
- 그래프 상단에는 경보 유형별 누적 건수를 표시하며 대상 인스턴스 집합만 합산한다.  
- 인스턴스별 경보 원형은 활성 경보가 없으면 정상 상태로 보이고 활성 경보가 있으면 강조 상태와 느낌표로 표시한다.  
- 인스턴스별 경보 원형의 강조 색상은 포함된 경보 중 최고 심각도(`Critical` 우선)로 결정한다.  
- 경보 레벨이 `Normal`로 수신되면 해당 경보의 활성 상태를 해제하고 강조 표시를 제거한다.  
- 서버 상태 복구 계열 이벤트는 대응되는 장애 상태를 해제하는 방식으로 처리한다.  
- 그리드에는 최신 경보가 상단에 추가되며 최대 50행까지만 유지한다.  
- 그리드의 값 표시는 경보 유형별 단위 규칙을 적용해 사용자 표시 형식으로 변환한다.  
- 그리드의 `Name` 열 클릭 시 경보 유형에 따라 전용 상세 화면으로 이동하거나 로그/스크립트 팝업을 연다.  
- 그리드의 `Log` 또는 `User Script` 열 클릭 시 인스턴스, 시각, 레벨과 함께 본문 내용을 팝업으로 표시한다.  
- `Resolve Alert`는 해제 가능한 경보에서만 활성화되며 실행 전에 확인 팝업을 노출한다.  
- 특정 옵션이 켜진 환경에서는 `Resolve Alert`를 DB 연결 실패 서버 경보에만 허용한다.  
- 알람 일시정지 등록 시 시간은 분 단위로 입력하며 허용 범위는 1분부터 10,080분까지다.  
- 알람 일시정지 등록과 해제 결과는 성공 또는 실패 메시지로 즉시 안내한다.  
- 한국어 환경에서는 우클릭 메뉴에 이벤트 설명 메뉴를 노출하고 `DB WAIT` 항목에서는 비활성화한다.  
- 비즈니스 그룹 모드에서는 인스턴스 관련 라벨과 목록 표시를 비즈니스명 기준으로 전환한다.  
- 대상 고정 모드로 열린 창에서는 상단 전환 영역을 숨기고 대상명을 포함한 제목으로 표시한다.  
  
## 제약 및 예외  
- 컴포넌트가 관리하는 인스턴스 또는 대상 필터에 맞지 않는 경보는 화면에 반영하지 않는다.  
- 초기 경보 저장 데이터가 준비되지 않은 상태에서는 수신 경보를 반영하지 않는다.  
- 레벨이 `Normal`인 이벤트는 신규 이력 행을 추가하지 않고 활성 경보 상태만 해제한다.  
- 일부 경보 유형은 값 컬럼을 비워 표시한다.  
- 메모 기능이 비활성화된 환경에서는 메모 아이콘과 메모 메뉴를 노출하지 않는다.  
- 알람 일시정지 아이콘은 관련 정책이 활성화된 환경에서만 노출한다.  
- 경보 상세 패널은 마우스가 패널을 벗어나거나 패널을 클릭하면 닫힌다.  
- 인스턴스 선택형 `1-Day History` 실행 시 대상을 선택하지 않으면 이동하지 않고 경고를 표시한다.  
  
## 비목표  
- 경보 임계치 계산과 탐지 엔진 로직은 이 기능의 범위에 포함하지 않는다.  
- 장기 보관 이력 분석이나 다일 기간 탐색은 이 기능의 범위에 포함하지 않는다.  
- 경보 해제, 일시정지, 메모의 서버 측 권한 정책 정의는 이 기능의 범위에 포함하지 않는다.  
- 외부 알림 채널 전송 정책은 이 기능의 범위에 포함하지 않는다.  
  
## 확인이 필요한 항목  
- 그리드 우클릭의 `Alarm Pause` 메뉴가 기본 숨김인데 상시 숨김 정책인지 조건부 노출 누락인지 확인이 필요하다.  
- 알람 일시정지 정책 조회가 메모 기능 옵션과 함께 켜졌을 때만 동작하는 현재 조건이 의도인지 확인이 필요하다.  
- `Alert log` 항목 클릭 시 별도 동작이 없는 현재 동작이 의도인지 확인이 필요하다.  
- `1-Day History` 이동 시 시간대 기준과 일부 알람명 매핑 규칙이 제품 정책과 일치하는지 확인이 필요하다.
