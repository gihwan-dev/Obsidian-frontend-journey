# 데이터 그리드의 우측 고정 컬럼과 리사이즈 동작 연구

다양한 React/JS 데이터 그리드 라이브러리들이 **고정/핀 컬럼(pinned or frozen columns)** 기능을 제공한다. 하지만 각 라이브러리마다 핀 컬럼의 리사이즈와 화면 적응에 대한 제약이 다르며, 우측 핀 컬럼을 지원하지 않거나 복잡한 예외 처리를 요구하는 경우도 있다. 아래는 주요 라이브러리들의 동작을 조사하여 정리한 결과다.

## 1. Material‑UI DataGrid (MUI X)

| 특징/제약 | 내용 |
|---|---|
| **핀 컬럼 기능** | DataGridPro에서 `pinnedColumns={{ left: [...], right: [...] }}` 형태로 왼쪽·오른쪽에 컬럼을 고정할 수 있다. 고정된 컬럼은 수평 스크롤 시 항상 보인다 [oai_citation:0‡mui.com](https://mui.com/x/react-data-grid/column-pinning/#:~:text=Pinned%20columns%20appearance). |
| **리사이즈** | 기본적으로 모든 컬럼은 드래그로 리사이즈 가능하며 각 컬럼 정의에 `resizable`, `minWidth`, `maxWidth`를 지정할 수 있다 [oai_citation:1‡mui.com](https://mui.com/x/react-data-grid/column-dimensions/#:~:text=Resizing). `onColumnResize` 및 `onColumnWidthChange` 이벤트로 리사이즈를 감지한다. |
| **우측 고정 시 이슈** | v7 이전에는 우측에 핀된 컬럼을 드래그로 리사이즈할 때 컬럼이 잠시 왼쪽으로 이동하였다가 마우스를 놓으면 다시 오른쪽으로 돌아가는 버그가 있었고, GitHub 이슈 #13548을 통해 수정되었다 [oai_citation:2‡github.com](https://github.com/mui/mui-x/issues/13548#:~:text=Current%20behavior). |
| **가이드라인** | MUI 문서에서는 핀 섹션의 폭이 전체 그리드 폭보다 크면 정상 동작을 보장하지 못하며, 오른쪽 컬럼을 핀하는 경우 같은 문제를 겪을 수 있으므로 너비를 조정해야 한다는 언급이 있다 [oai_citation:3‡mui.com](https://mui.com/x/react-data-grid/column-pinning/#:~:text=Pinned%20columns%20appearance). |

**정리:** MUI X는 좌·우 고정 컬럼 모두 지원하지만, **핀 섹션의 합계 너비가 그리드 폭을 초과하지 않아야** 정상적으로 작동한다. 리사이즈는 기본적으로 각 컬럼 오른쪽에 드래그 핸들이 있으며, 버그 수정 이후 우측 컬럼에서도 정상적으로 동작하지만 폭 제한이 필요하다.

## 2. AG Grid

| 특징/제약 | 내용 |
|---|---|
| **핀 컬럼** | `pinned: 'left'` 또는 `pinned: 'right'` 속성으로 컬럼을 왼쪽·오른쪽에 고정할 수 있다. 핀 컬럼은 중앙 영역과 별도로 스크롤되지 않는다. |
| **핀 섹션 폭 제약** | 문서에는 "핀된 영역(left/right)의 크기는 그리드 폭 – 50px 이하로 제한된다"고 설명한다 [oai_citation:4‡ag-grid.com](https://www.ag-grid.com/javascript-data-grid/column-pinning/#:~:text=Resizing%20Pinned%20Sections%20Copy%20Link). 즉, **핀 영역이 너무 넓어지면 중앙 영역에 최소 50px 이상 공간을 남겨야 한다**. |
| **자동 unpin** | 핀 영역이 너무 넓어지면 AG Grid는 **왼쪽 핀 영역에서부터 컬럼을 자동으로 unpin** 하여 중앙 영역을 확보한다 [oai_citation:5‡github.com](https://github.com/ag-grid/ag-grid/issues/8335#:~:text=Current%20behavior%20I%20noticed%20an,6%2C7%2C8%2C9%2C10%5D%20which%20are%20scrollable). 2024년 이슈 #8335에 따르면 이 동작으로 컬럼 순서가 바뀌는 불편이 있어, unpin 방향을 바꿀 수 있도록 하는 `processUnpinnedColumns` 콜백이 제공된다 [oai_citation:6‡github.com](https://github.com/ag-grid/ag-grid/issues/8335#:~:text=Current%20behavior%20I%20noticed%20an,6%2C7%2C8%2C9%2C10%5D%20which%20are%20scrollable). 라이브러리는 기본적으로 왼쪽에서부터 unpin하며, 개발자가 오른쪽부터 unpin하도록 커스터마이징 할 수 있다 [oai_citation:7‡liveloveapp.com](https://liveloveapp.com/blog/2025-02-14-ag-grid-pinned-columns#:~:text=Out%20the%20box%2C%20AG%20Grid,width%20for%20the%20center%20viewport). |
| **리사이즈** | 컬럼 리사이즈는 기본적으로 지원되며, `colResizeDefault` 값에 따라 인접 컬럼에 폭을 분배하는 방식이 바뀐다. `Shift` 키를 누른 채 리사이즈하면 반대편 컬럼이 움직이는 동작이 있는데, 이 동작이 우측 핀 컬럼에서 어색하게 느껴진다는 이슈(#4294)가 제기되었다 [oai_citation:8‡github.com](https://github.com/ag-grid/ag-grid/issues/4294#:~:text=%2A%20Hold%20down%20,resizing%20behavior%20with%20pinned%20columns). AG Grid 팀은 버그가 아니며, 필요하면 기능 요청으로 처리해야 한다고 답했다 [oai_citation:9‡github.com](https://github.com/ag-grid/ag-grid/issues/4294#:~:text=More%20actions). |

**정리:** AG Grid는 핀 컬럼의 폭이 커지면 **자동으로 일부 컬럼을 unpin** 하여 중앙 영역을 확보한다. 우측 핀 컬럼도 지원하지만, 폭 제약과 unpin 로직 때문에 컬럼 순서가 바뀔 수 있으므로 `processUnpinnedColumns`로 제어해야 한다. 리사이즈 시 Shift 동작 등 복잡도가 있으니 사용자가 예측 가능한 UX를 제공하는 것이 중요하다.

## 3. Kendo UI Grid (Telerik)

| 특징/제약 | 내용 |
|---|---|
| **고정 컬럼** | Telerik Grid의 `locked` 속성으로 컬럼을 고정할 수 있지만 **좌측으로만 고정 가능**하다 [oai_citation:10‡telerik.com](https://www.telerik.com/kendo-jquery-ui/documentation/controls/grid/columns/locked-columns#:~:text=For%20a%20runnable%20example%2C%20refer,locked%20columns%20in%20the%20Grid). 우측 고정은 지원하지 않는다. |
| **폭 제한** | 문서에서는 "고정 컬럼의 총 너비는 그리드 너비 minus (스크롤바 너비 × 3)보다 작아야 하며, 최소한 하나의 비고정 컬럼이 표시되어야 한다"고 명시한다 [oai_citation:11‡telerik.com](https://www.telerik.com/kendo-jquery-ui/documentation/controls/grid/columns/locked-columns#:~:text=For%20a%20runnable%20example%2C%20refer,locked%20columns%20in%20the%20Grid). |
| **리사이즈** | 컬럼 리사이즈는 가능하지만, 고정 컬럼은 좌측에만 존재하므로 우측 고정 문제는 발생하지 않는다. |

**정리:** Kendo UI는 우측 고정 컬럼 자체를 지원하지 않는다. 고정 컬럼의 합계 폭을 제한하여 중앙 영역의 스크롤을 확보한다.

## 4. Syncfusion Blazor/React DataGrid

| 특징/제약 | 내용 |
|---|---|
| **고정 컬럼(frozen)** | `IsFrozen="true"`와 `Freeze="Right"` 속성으로 우측에 컬럼을 고정할 수 있다 [oai_citation:12‡blazor.syncfusion.com](https://blazor.syncfusion.com/documentation/datagrid/frozen-column#:~:text=Add%20or%20remove%20frozen%20columns,by%20dragging%20the%20column%20separator). `Left`, `Right`, `Fixed` 옵션이 있으며, `AllowFreezeLineMoving=true`를 설정하면 사용자가 구분선을 드래그하여 동적으로 고정 컬럼 수를 늘리거나 줄일 수 있다. |
| **리사이즈** | 문서에서는 컬럼 리사이즈(`AllowResizing=true`)와 `MinWidth`, `MaxWidth` 설정을 지원하지만 [oai_citation:13‡blazor.syncfusion.com](https://blazor.syncfusion.com/documentation/datagrid/column-resizing#:~:text=To%20enable%20column%20resizing%2C%20set,the%20Grid%20component%20to%20true), 고정 컬럼의 리사이즈에 대한 구체적인 설명은 없었다. 그러나 고정 컬럼을 추가/제거할 때는 높이 계산, 필터·그룹·행 높이 등 여러 기능과 호환성 문제가 있다는 제한 사항을 안내한다 [oai_citation:14‡blazor.syncfusion.com](https://blazor.syncfusion.com/documentation/datagrid/frozen-column#:~:text=Limitations%20of%20frozen%20columns%20and,freeze%20direction). |

**정리:** Syncfusion DataGrid는 좌·우 고정 컬럼을 지원한다. 사용자가 고정 영역의 분리선을 움직여 위치를 조정할 수 있지만, 리사이즈와 관련된 제약은 명확히 문서화되어 있지 않다. 고정 컬럼을 넓히면 중앙 영역이 좁아지므로 개발자가 폭 제한을 걸어야 한다.

## 5. PrimeReact DataTable

| 특징/제약 | 내용 |
|---|---|
| **고정 컬럼(Frozen)** | 각 컬럼에 `frozen` 속성을 주고 `alignFrozen="left"` 또는 `alignFrozen="right"`를 지정하여 컬럼을 좌측·우측에 고정할 수 있다 [oai_citation:15‡primereact.org](https://primereact.org/datatable/#:~:text=Frozen%20Columns). 예제에서는 우측에 잔액 컬럼을 고정하는 토글 버튼을 제공한다. |
| **리사이즈** | DataTable은 컬럼 리사이즈, 재정렬, 그룹화 등을 지원하지만 문서에 **고정 컬럼과 리사이즈의 상호작용에 대한 설명이 없다**. 실제 동작에서는 리사이즈 시 스크롤 영역이 발생하지만, 우측 고정 컬럼이 늘어나면 중앙 영역이 좁아지는 문제가 발생할 수 있다. |

**정리:** PrimeReact는 우측 고정 컬럼을 지원하지만, 리사이즈와의 정확한 상호작용은 명확히 문서화되어 있지 않다. 사용자가 고정 컬럼을 크게 늘리면 중앙 영역을 침범할 수 있으므로 폭 제한이 필요하다.

## 6. Mantine React Table (MRT)

| 특징/제약 | 내용 |
|---|---|
| **핀 컬럼** | TanStack Table을 기반으로 하며 `enablePinning` 옵션과 `columnPinning: { left: [], right: [] }` 상태로 좌·우 컬럼을 핀할 수 있다 [oai_citation:16‡mantine-react-table.com](https://www.mantine-react-table.com/docs/guides/column-pinning#:~:text=Column%20pinning%20can%20simply%20be,true). |
| **컬럼 리사이즈** | `enableColumnResizing` 옵션으로 리사이즈를 활성화하고, `columnResizeMode: 'onChange' | 'onEnd'` 등으로 동작 방식을 결정한다 [oai_citation:17‡mantine-react-table.com](https://www.mantine-react-table.com/docs/guides/column-resizing#:~:text=1columnResizeMode). |
| **핀 컬럼과 리사이즈의 관계** | 공식 문서에는 핀 컬럼과 리사이즈의 상호작용에 대한 구체적인 설명이 없다. 일반적으로 TanStack Table은 핀 컬럼을 중앙 영역과 분리하여 렌더링하므로, 리사이즈 시 핀 섹션의 폭 증가가 중앙 영역의 폭 감소로 이어진다. 폭 제한을 개발자가 직접 관리해야 한다. |

**정리:** Mantine React Table은 우측 핀 컬럼을 지원하지만, 리사이즈와 관련된 제약이 문서화되어 있지 않다. 개발자가 `columnPinning` 상태와 `columnSizing` 상태를 동기화해 폭을 제한해야 한다.

## 7. Tabulator

| 특징/제약 | 내용 |
|---|---|
| **Frozen Columns** | `frozen: true` 속성을 사용해 좌측·우측 컬럼을 고정할 수 있으며, 고정 컬럼은 **컬럼 정의 배열에서 연속적으로 배치**해야 한다 [oai_citation:18‡tabulator.info](https://tabulator.info/docs/6.3/layout#:~:text=You%20can%20freeze%20the%20position,the%20table%20is%20scrolled%20horizontally). 고정 컬럼은 드래그로 순서를 변경할 수 없고, **최소한 하나의 비고정 컬럼이 필요**하다 [oai_citation:19‡tabulator.info](https://tabulator.info/docs/6.3/layout#:~:text=Note%3A%20%20Frozen%20columns%20are,and%20dropping%20the%20column%20header). |
| **리사이즈** | `resizable` 옵션으로 컬럼 리사이즈를 제어하며, `resizableColumnFit:true`를 사용하면 한 컬럼을 늘릴 때 인접 컬럼이 반대로 줄어들어 전체 너비를 일정하게 유지한다 [oai_citation:20‡tabulator.info](https://tabulator.info/docs/6.3/layout#:~:text=Resizable%20Columns%20%20%20121). `minWidth`와 `maxWidth`로 폭을 제한할 수 있다 [oai_citation:21‡tabulator.info](https://tabulator.info/docs/6.3/layout#:~:text=all%20the%20columns%20will%20change). |
| **이슈** | 2023년 이슈 #4331에서는 여러 개의 고정 컬럼 중 **마지막 컬럼만 리사이즈 핸들이 나타나고 다른 고정 컬럼은 리사이즈가 되지 않는 버그**가 보고되었다 [oai_citation:22‡github.com](https://github.com/olifolkerd/tabulator/issues/4331#:~:text=Hello%2C%20I%20have%20a%20table,all%20columns%20are%20resizable). 개발자는 z-index 문제를 수정하겠다고 답했으며, Tabulator 5.6 버전에서 수정되었다 [oai_citation:23‡github.com](https://github.com/olifolkerd/tabulator/issues/4331#:~:text=Hey%20%40schlundus). |

**정리:** Tabulator는 우측 고정 컬럼을 지원하지만, 고정 컬럼이 연속적이어야 하고 최소 하나의 일반 컬럼을 남겨야 한다는 제약이 있다. `resizableColumnFit` 옵션을 사용하면 리사이즈 시 전체 폭을 유지할 수 있다. 특정 버전에서 리사이즈 핸들이 안 보이는 버그가 있었으므로 최신 버전을 사용하는 것이 좋다.

## 8. 기타 라이브러리 및 커뮤니티 사례

- **DevExtreme**: React DataGrid 데모에서 컬럼 고정(fixing) 기능을 제공하지만, 우측 고정 및 리사이즈 동작에 대한 명시적인 설명은 찾기 어려웠다.
- **Infragistics Ignite UI for React**: `pinned="left"`/`"right"` 속성으로 컬럼을 고정할 수 있다. 공식 문서는 핀 컬럼과 리사이즈의 상호작용에 대해 언급하지 않지만, 예제에서 오른쪽에 여러 컬럼을 고정하는 모습은 보여 준다 [oai_citation:24‡infragistics.com](https://www.infragistics.com/products/ignite-ui-react/react/components/grids/data-grid/column-pinning#:~:text=React%20Grid%20Column%20Pinning%20Overview).

## 공통 패턴 및 구현 지침

1. **우측 고정 컬럼은 상대적으로 복잡하다.** 많은 라이브러리가 좌측 고정만 지원하거나, 우측 고정을 지원하더라도 문서화가 부족하여 버그나 일관성 문제가 보고된다.
2. **핀 섹션 폭 제한 필요:** AG Grid와 Kendo UI는 고정 컬럼의 합계 폭이 그리드 폭을 넘지 않도록 제한한다 [oai_citation:25‡ag-grid.com](https://www.ag-grid.com/javascript-data-grid/column-pinning/#:~:text=Resizing%20Pinned%20Sections%20Copy%20Link) [oai_citation:26‡telerik.com](https://www.telerik.com/kendo-jquery-ui/documentation/controls/grid/columns/locked-columns#:~:text=For%20a%20runnable%20example%2C%20refer,locked%20columns%20in%20the%20Grid). 우측 고정 컬럼을 리사이즈할 때 중앙 영역에 최소한의 스크롤 여유를 남겨야 한다.
3. **자동 unpin 또는 제약:** AG Grid처럼 고정 컬럼이 많을 경우 자동으로 일부 컬럼을 unpin하는 기능을 제공하는 라이브러리도 있다 [oai_citation:27‡github.com](https://github.com/ag-grid/ag-grid/issues/8335#:~:text=Current%20behavior%20I%20noticed%20an,6%2C7%2C8%2C9%2C10%5D%20which%20are%20scrollable). 하지만 순서가 변경될 수 있으므로 개발자가 `processUnpinnedColumns` 같은 콜백으로 제어해야 한다 [oai_citation:28‡liveloveapp.com](https://liveloveapp.com/blog/2025-02-14-ag-grid-pinned-columns#:~:text=Out%20the%20box%2C%20AG%20Grid,width%20for%20the%20center%20viewport).
4. **리사이즈 핸들 위치:** 대부분의 그리드는 컬럼 오른쪽 경계에서 리사이즈를 수행한다. 우측 고정 컬럼을 좌측으로 리사이즈하게 만들려면 별도의 예외 처리(리사이즈 핸들 위치 변경)가 필요하지만, 대부분의 라이브러리는 이를 지원하지 않는다. 대신 핀 컬럼의 폭을 제한하거나 리사이즈를 비활성화하는 것이 일반적이다.
5. **버그 가능성:** MUI나 Tabulator 사례처럼 우측 고정 컬럼에서 리사이즈 시 위치가 이동하거나, 핸들이 안 보이는 버그가 보고되었다 [oai_citation:29‡github.com](https://github.com/mui/mui-x/issues/13548#:~:text=Current%20behavior) [oai_citation:30‡github.com](https://github.com/olifolkerd/tabulator/issues/4331#:~:text=Hello%2C%20I%20have%20a%20table,all%20columns%20are%20resizable). 최신 버전을 사용하고, 리사이즈 이벤트를 테스트해야 한다.

## 구현을 위한 제안

- **우측 고정 컬럼을 반드시 지원해야 하는지 검토:** 많은 라이브러리가 좌측 고정만 지원하거나 우측 고정에서 문제가 발생한다. 사용자 경험을 고려해 좌측 고정으로만 제한하거나, 우측 고정 컬럼은 리사이즈를 비활성화하는 선택지도 있다.
- **폭 제한/스크롤 영역 확보:** 우측 고정 컬럼 리사이즈 시 중앙 영역이 너무 좁아지는 것을 방지하려면, **고정 섹션의 최대 폭을 전체 테이블 폭의 일정 비율이나 픽셀 값으로 제한**하는 로직을 구현한다. AG Grid처럼 여유 공간이 없으면 일부 컬럼을 자동으로 일반 컬럼으로 이동시키는 것도 방법이다.
- **리사이즈 핸들 위치 일관성:** 대부분의 그리드가 **컬럼 오른쪽 경계**에서 리사이즈를 수행하기 때문에, 우측 고정 컬럼만 좌측 경계에서 리사이즈하도록 만드는 것은 사용자에게 혼란을 줄 수 있다. 한 방향(예: 항상 오른쪽)으로 리사이즈하도록 제약하는 것이 구현 복잡도를 줄인다.
- **예외 처리:** 우측 고정 컬럼에서 리사이즈를 허용하지 않거나, 리사이즈할 경우 자동으로 이전에 비고정 컬럼 중 하나를 핀 영역으로 이동시키는 등 **명확한 제약 조건**을 문서화하여 사용자에게 알린다.
- **기능 토글 제공:** 사용자가 우측 고정 컬럼 리사이즈를 자주 사용하지 않을 수 있으므로, 설정 옵션으로 리사이즈 가능 여부를 제어하도록 한다. 예: `allowRightFixedResize: false` 처럼 우측 핀 컬럼은 너비를 변경할 수 없도록 한다.

## 결론

데이터 그리드 라이브러리들은 우측 고정 컬럼과 리사이즈 동작을 제각기 다르게 처리하며, 명확한 표준이 없는 상황이다. **가장 일반적인 패턴은 핀된 섹션의 총 너비가 그리드 너비를 초과하지 않도록 제한**하고, 필요하면 일부 컬럼을 자동으로 해제(unpin)하는 것이다. 우측 고정 컬럼에 대해 리사이즈 핸들을 좌측으로 옮기는 기능을 구현하려면 사용자 경험과 기술적 복잡성을 고려해야 하며, 많은 라이브러리들이 이를 지원하지 않으므로 **폭 제한이나 리사이즈 비활성화가 현실적인 해결책**이 될 수 있다.
