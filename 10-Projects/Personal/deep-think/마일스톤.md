# 🚀 데스크탑 AI 채팅 앱 전체 마일스톤 (v1.0 릴리즈 로드맵)

### 🎯 Milestone 1: 인프라 뼈대 및 Mock 파이프라인 개통 (Foundation)

가장 먼저 프론트(React)와 백엔드(Rust)가 뚫려 있는지 확인하는 "고속도로 포장" 단계입니다.

- **Task 1.1:** Tauri v2 + React/Vite 초기화 및 FSD 디렉토리 구조 셋업.
    
- **Task 1.2:** Zustand(전역) 및 Jotai(스트리밍) 스토어 보일러플레이트 작성.
    
- **Task 1.3:** 브라우저 환경에서 동작하는 UI 컴포넌트 검증을 위해 **Vitest Browser Mode** 테스트 환경 구축.
    
- **Task 1.4:** Rust 백엔드에서 가짜(Mock) 토큰을 생성해 Tauri IPC로 프론트엔드에 스트리밍하는 단방향 통신 구현.
    

### 🎯 Milestone 2: Tool 서버 및 단일 에이전트 연동 (Core BE)

이제 가짜 데이터를 빼고, 진짜 CLI 에이전트(Codex 또는 Claude Code)를 백엔드에 붙입니다.

- **Task 2.1:** Rust 백엔드에 **기본 제공 Tool**을 먼저 정의하고, 이를 제공하는 **DeepThink Tools MCP Server** 구현. (프로젝트 루트 샌드박스/allowlist/timeout 포함)
    
- **Task 2.2:** Codex/Claude Code 워커 프로세스를 실행하는 **Agent Driver** 구현. (stdout/stderr 스트리밍 파싱 + Tool 서버 연동)
    
- **Task 2.3:** `send_message` / `cancel_run` / `agent_update`까지 End-to-End 연결. (단일 워커 기준 기본 채팅 + Cancel + Tool 실행 로그가 UI에 보임)
    

### 🎯 Milestone 3: Deep Think UI 및 미시 상태 관리 (Core FE)

백엔드에서 쏟아지는 복잡한 추론 상태를 화면에 예쁘게 그리는 프론트엔드 집중 단계입니다.

- **Task 3.1:** Jotai Atom을 활용해 단일 메시지 내의 상태(생각 중, 도구 실행 중, 완료)를 컴포넌트 단위로 격리하여 리렌더링 최적화.
    
- **Task 3.2:** '사고 과정 아코디언(Thought Process Tree)' UI 컴포넌트 구현. (D2Coding 폰트 전역 적용)
    
- **Task 3.3:** 백엔드의 **Tool 실행 이벤트(시작/진행/종료)** 를 수신하여 🛠️ 도구 실행 뱃지 및 터미널 뷰포트 UI에 상태 반영.
    

### 🎯 Milestone 4: MoA (Multi-Agent) 오케스트레이션 및 딥씽크 구현 (The Brain)

이 프로젝트의 꽃입니다. 단일 에이전트를 넘어, 여러 에이전트가 지지고 볶으며 정답을 찾는 로직을 백엔드에 구현합니다.

- **Task 4.1:** 사용자 질문 시, Rust 백엔드에서 각기 다른 프롬프트(성능, 안정성 등)를 가진 3개의 워커(Worker) 세션을 병렬(tokio::spawn)로 실행.
    
- **Task 4.2:** 각 워커가 자율적으로 도구를 사용하고 에러 발생 시 스스로 교정(Reflexion)하는 내부 루프(최대 재시도 횟수 제한) 구현.
    
- **Task 4.3:** 워커들의 실행 결과를 취합하여, 심판(Judge) 에이전트에게 최종 병합과 'Thought Summary' 작성을 지시하는 파이프라인 완성.
    

### 🎯 Milestone 5: 렌더링 성능 최적화 및 고도화 (Optimization)

대화가 길어지고 로그가 쌓일 때 FPS가 떨어지는 것을 막는 퍼포먼스 튜닝 단계입니다.

- **Task 5.1:** `@tanstack/react-virtual`을 도입하여 채팅 스레드의 가상화(Virtualization) 렌더링 구현 및 DOM 노드 수 프로파일링.
    
- **Task 5.2:** 터미널 출력 결과나 방대한 빌드 로그를 React 상태에서 분리하여 `xterm.js` (Canvas 기반)로 렌더링하도록 전환.
    
- **Task 5.3:** 창 크기 조절, 부드러운 자동 스크롤, 에이전트 폭주 시 '명시적 중단(Cancel)' 인터랙션 구현.
    

### 🎯 Milestone 6: CI/CD 파이프라인 구축 및 릴리즈 (Automation)

개발된 데스크탑 앱을 패키징하고 자동 배포하는 마무리 단계입니다.

- **Task 6.1:** Changesets를 활용한 버전 관리 및 변경 사항(Changelog) 자동화 셋업.
    
- **Task 6.2:** 무료 버전의 Self-Hosted GitLab 환경에 맞춰 `.gitlab-ci.yml` 스크립트 작성 (리눅스/윈도우/맥OS 크로스 컴파일 및 Tauri 빌드).
    
- **Task 6.3:** 릴리즈 아티팩트 생성 및 최종 E2E 테스트 통과 확인 후 v1.0 릴리즈.
