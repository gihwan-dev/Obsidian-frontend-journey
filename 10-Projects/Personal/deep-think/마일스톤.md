# 🚀 데스크탑 AI 채팅 앱 전체 마일스톤 (v1.0 릴리즈 로드맵)

### 🎯 Milestone 1: 인프라 뼈대 및 Mock 파이프라인 개통 (Foundation)

가장 먼저 프론트(React)와 백엔드(Rust)가 뚫려 있는지 확인하는 "고속도로 포장" 단계입니다.

- **Task 1.1:** Tauri v2 + React/Vite 초기화 및 FSD 디렉토리 구조 셋업.
    
- **Task 1.2:** 전역 상태(Zustand) 최소 스켈레톤 구성. (에이전트/워크스페이스/설정 등)
    
- **Task 1.3:** 스트리밍 미시 상태(Jotai) 최소 스켈레톤 구성. (메시지/워커 단위 스트림)
    
- **Task 1.4:** 브라우저 환경에서 동작하는 UI 컴포넌트 검증을 위해 **Vitest Browser Mode** 테스트 환경 구축.
    
- **Task 1.5:** Rust 백엔드에서 가짜(Mock) 이벤트/토큰을 생성해 Tauri IPC로 프론트엔드에 스트리밍하는 단방향 통신 구현.
    

### 🎯 Milestone 2: Tool 서버 및 단일 에이전트 연동 (Core BE)

이제 가짜 데이터를 빼고, 진짜 CLI 에이전트(Codex 또는 Claude Code)를 백엔드에 붙입니다.

- **Task 2.1:** **DeepThink Tools MCP Server** 골격 + 프로젝트 루트 샌드박스 적용. (읽기 위주 Tool부터 시작)
    
- **Task 2.2:** 쓰기 최소 기능 추가. (예: patch 기반 수정)
    
- **Task 2.3:** 쉘 실행 최소 기능 추가. (allowlist/timeout 포함)
    
- **Task 2.4:** 단일 워커 실행(Agent Driver) 구현: 우선 Codex 1개로 시작. (스트리밍 수신/정규화)
    
- **Task 2.5:** End-to-End 연결: `send_message` → 스트리밍 응답 → `agent_update` 이벤트로 FE 반영.
    
- **Task 2.6:** Tool 호출/결과를 이벤트로 정규화하여 FE에서 “도구 실행 내역”이 보이게 연결.
    
- **Task 2.7:** Cancel 연결: `cancel_run` 시 워커/하위 작업 중단 + UI 상태 반영.
    
- **Task 2.8:** Claude Code 연결 추가(옵션) 또는 에이전트 선택 스위치 제공.
    

### 🎯 Milestone 3: Deep Think UI 및 미시 상태 관리 (Core FE)

백엔드에서 쏟아지는 복잡한 추론 상태를 화면에 예쁘게 그리는 프론트엔드 집중 단계입니다.

- **Task 3.1:** 이벤트 수신 파이프라인(backend → store) 고정 및 메시지/워커 단위 상태 모델 확정.
    
- **Task 3.2:** 병렬 워커 뷰 기본 구현. (탭/카드로 워커별 스트리밍 분리)
    
- **Task 3.3:** Thought Tree(아코디언) 기본 구현. (Progressive Disclosure + D2Coding 적용)
    
- **Task 3.4:** Tool 실행 UI 기본 구현. (뱃지 + 간단 로그 뷰어로 시작)
    
- **Task 3.5:** Cancel UI 기본 구현. (전체/워커별 중단, 상태 뱃지)
    

### 🎯 Milestone 4: MoA (Multi-Agent) 오케스트레이션 및 딥씽크 구현 (The Brain)

이 프로젝트의 꽃입니다. 단일 에이전트를 넘어, 여러 에이전트가 지지고 볶으며 정답을 찾는 로직을 백엔드에 구현합니다.

- **Task 4.1:** 멀티 워커 실행 골격 구현. (N개 워커 병렬 실행 + 워커별 이벤트 분리)
    
- **Task 4.2:** Thinking Level(Quick/Standard/Deep)과 워커 수/시간/도구 예산 매핑 규칙 확정 및 적용.
    
- **Task 4.3:** 결과 취합(기본 병합) 구현: 워커 결과를 모아 Final Answer 생성.
    
- **Task 4.4:** Judge 단계 추가: 워커 결과를 평가/병합하고 Thought Summary 생성.
    
- **Task 4.5:** Reflexion(자기 교정) 루프 추가(옵션): 실패 시 제한된 재시도/수정.
    

### 🎯 Milestone 5: 렌더링 성능 최적화 및 고도화 (Optimization)

대화가 길어지고 로그가 쌓일 때 FPS가 떨어지는 것을 막는 퍼포먼스 튜닝 단계입니다.

- **Task 5.1:** `@tanstack/react-virtual`을 도입하여 채팅 스레드의 가상화(Virtualization) 렌더링 구현 및 DOM 노드 수 프로파일링.
    
- **Task 5.2:** 터미널 출력 결과나 방대한 빌드 로그를 React 상태에서 분리하여 `xterm.js` (Canvas 기반)로 렌더링하도록 전환.
    
- **Task 5.3:** 자동 스크롤(핀/해제) 및 긴 스트리밍 UX 안정화.
    
- **Task 5.4:** 창 크기 조절 및 레이아웃/리사이즈 성능 고도화.
    
- **Task 5.5:** Cancel UX 고도화. (워커별 Stop, 폭주 감지/안내 등)
    

### 🎯 Milestone 6: CI/CD 파이프라인 구축 및 릴리즈 (Automation)

개발된 데스크탑 앱을 패키징하고 자동 배포하는 마무리 단계입니다.

- **Task 6.1:** Changesets를 활용한 버전 관리 및 변경 사항(Changelog) 자동화 셋업.
    
- **Task 6.2:** Self-Hosted GitLab에 맞춘 기본 CI 파이프라인 구성. (테스트/빌드 + 단일 OS 아티팩트)
    
- **Task 6.3:** CI 확장: 캐시/매트릭스 적용 및 크로스 플랫폼 빌드로 확장.
    
- **Task 6.4:** 릴리즈 아티팩트 생성 및 최종 E2E 테스트 통과 확인 후 v1.0 릴리즈.
