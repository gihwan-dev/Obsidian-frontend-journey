## Exercise 1.6 — `if`는 함수가 아니다

- 조건부 표현식은 단순한 함수 호출이 아님
- `if`는 **구문형(syntactic form)** 이며 특별한 평가 규칙을 가짐
- 자바스크립트는 **인수 우선 평가(applicative-order evaluation)** 를 사용함
- 함수 호출 시 모든 인수가 먼저 평가됨

```js
function new_if(predicate, then_clause, else_clause) {  
  return predicate ? then_clause : else_clause;  
}
```

이것을 `sqrt_iter`에 적용하면:

```js
return new_if(  
  is_good_enough(guess, x),  
  guess,  
  sqrt_iter(improve(guess, x), x)  
);
```

- 세 번째 인수 `sqrt_iter(...)` 가 조건 검사 전에 먼저 평가됨
- 그 결과 재귀가 멈추지 않음
- 무한 재귀 발생

### 핵심

> 조건부는 함수가 아니라 **평가 규칙을 가진 특수 구문형**이다.

이는 다음을 보여준다:

- 평가 전략은 프로그램의 의미를 결정한다.
- 구문형과 함수는 근본적으로 다르다.
