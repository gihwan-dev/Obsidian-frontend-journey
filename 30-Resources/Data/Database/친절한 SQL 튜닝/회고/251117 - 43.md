SQL은 **Structured Query Language**의 줄임말로 기본적이고 구조적이고 집합적이며 선언적인 질의어 입니다. 그래서 이를 분석해 실행 가능한 형태의 프로시저로 변환할 필요가 있습니다. 이 프로시저를 작성하고 컴파일해서 실행가능한 형태로 만드는 전 과정을 SQL 최적화 라고 한다고 합니다.

특별히 재밌었던 내용은 실행 계획(Execution Plan)에 대한 내용 이었습니다! SQL 옵티마이저는 **자동차 네비게이션**과 여러모로 흡사하다고 합니다. "SQL 실행 결로"를 파악해 가장 효율적인 경로를 선택하기 때문입니다. 그리고 이러한 SQL 실행 경로를 미리보는 기능을 **실행 계획(Execution Plan)**이라고 합니다.

SQL 옵티마이저가 생성한 처리 절차를 트리구조로 표현한 것이며 SQL이 테이블을 스캔하는지, 인덱스를 스캔하는지 등등을 파악할 수 있다고 합니다. **실행 계획(Execution Plan)**의 비용(Cost)은 쿼리를 수행하는 동안 발생할 것으로 예상하는 I/O 횟수 또는 예상 소요시간을 표현한 값이라고 하네요!
