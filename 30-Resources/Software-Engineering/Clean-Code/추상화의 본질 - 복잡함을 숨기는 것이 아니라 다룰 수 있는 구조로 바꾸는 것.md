# 추상화의 본질: 복잡함을 숨기는 것이 아니라 다룰 수 있는 구조로 바꾸는 것

> [!quote] 핵심 문장
> 추상화의 핵심은 복잡함을 숨기는 것이 아니라, 복잡한 문제를 다룰 수 있는 구조로 바꾸는 데 있다.

## 핵심 논지

추상화는 흔히 **복잡한 내부를 감추고 인터페이스를 단순하게 만드는 것**으로 설명된다. 이 설명은 맞지만, 이것만으로는 추상화의 본질을 충분히 설명하지 못한다. 단순히 복잡함을 가리는 것만으로는 추상화가 왜 중요한지, 왜 유용한지 설명이 부족하다.

추상화의 핵심은 **복잡한 문제를 여러 부분 문제로 나누는 과정**과 결합될 때 비로소 드러난다.

- 하나의 큰 문제를 작은 문제들로 **분해**하고
- 각각의 부분 문제를 함수나 모듈처럼 **독립적인 단위**로 다룰 수 있게 만들 때

추상화는 실제적인 힘을 가진다.

> 추상화는 단순히 복잡함을 숨기는 기술이 아니라, **문제 분해의 결과를 구조화하고 재사용 가능하게 만드는 기술**이다.

## 어떻게 나누느냐가 추상화의 품질을 결정한다

같은 문제라도 분해 방식에 따라 추상화의 품질이 달라진다.

### 예시: 라면을 끓여 먹는다

**높은 추상화 수준으로 나누기:**

```
준비한다 → 끓인다 → 먹는다
```

**낮은 추상화 수준으로 나누기:**

```
재료를 꺼낸다 → 물을 끓인다 → 재료를 넣는다 → 익힌다 → 먹는다
```

핵심 질문: **어디까지를 하나의 부분 문제로 볼 것인가?**

- "라면을 끓인다"를 더 이상 나누지 않으면 → "된장국을 끓인다" 같은 비슷한 문제에서 **중복이 발생**한다.
- "재료를 준비한다", "무언가를 끓인다", "무언가를 먹는다"처럼 **일반화된 단위**로 추상화하면 → 서로 다른 문제에서도 **공통 구조를 활용**할 수 있다.

## 좋은 추상화의 조건

1. 복잡한 문제를 **적절한 부분 문제**로 나눈다
2. 각 부분 문제가 **올바른 책임**과 **적절한 추상화 수준**을 가진 단위가 된다
3. 추상화 수준이 서로 **뒤섞이지 않고 정렬**되어 있다
4. 각 단위는 다른 맥락에서도 **재사용될 수 있을 만큼 일반적**이다

## 글에 쓸 만한 핵심 문장들

- 추상화의 핵심은 복잡함을 숨기는 것이 아니라, 복잡한 문제를 다룰 수 있는 구조로 바꾸는 데 있다.
- 좋은 추상화는 좋은 문제 분해에서 시작된다.
- 추상화의 가치는 감춤 자체가 아니라 재사용 가능한 부분 문제를 만드는 데서 나온다.
- 어떻게 나누느냐가 결국 어떤 추상화를 만들 것이냐를 결정한다.
- 올바른 추상화는 적절한 책임 분리와 정렬된 추상화 수준 위에서 성립한다.
