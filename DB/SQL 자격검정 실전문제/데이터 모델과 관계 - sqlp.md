### 성능 데이터 모델링이란?
데이터베이스 성능 향상을 목적으로 설계단계의 데이터 모델링 때부터 성능과 관련된 사항이 데이터 모델링에 반영될 수 있도록 하는 것

- 데이터의 증가가 빠를수록 성능저하에 따른 성능개선비용은 증가
- 데이터모델은 성능을 튜닝하면서 변경이 될 수 있는 특징이 있음
- 분석/설계 단계에서 성능을 고려한 데이터모델링을 수행할 경우 성능저하에 따른 Rework 비용을 최소화 할 수 있는 기회를 가지게 됨


### 성능 데이터 모델링 수행 절차
1. 데이터모델링을 할 때 정규화를 정확하게 수행
2. 데이터베이스 용량산정을 수행
3. 데이터베이스에 발생되는 트랜잭션의 유형을 파악
4. 용량과 트랜잭션의 유형에 따라 반정규화를 수행
5. 이력모델의 조정, PK/FK 조정, 슈퍼타입/서브타입 조정 등을 수행
6. 성능관점에서 데이터모델을 검증

### 성능 데이터 모델링 고려 사항
- 중복된 데이터를 제거함으로써 조회성능을 향상시킬 수 있음
- 용량산정은 전체적인 데이터베이스에 발생되는 트랜잭션의 유형과 양을 분석하는 자료가 되므로 성능데이터 모델링을 할 때 중요한 작업이 될 수 있음
- 물리적인 데이터 모델링을 할 때 PK/FK의 칼럼의 순서조정, FK 인덱스 생성 등은 성능 향상을 위한 데이터 모델링 작업에 중요한 요소가 됨
- 이력데이터는 시간에 따라 반복적으로 발생이 되기 때문에 대량 데이터일 가능성이 높아 특별히 성능을 고려해 칼럼 등을 추가하도록 설계해야 한다

### 정규화와 비 정규화
#### 정규화
##### 정의
데이터의 중복을 최소화하고 이상 현상을 제거하기 위해 테이블을 분해하는 과정

##### 목적
1. 데이터 중복 최소화 - 저장 공간 절약
2. 이상 현상 방지
	1. 삽입 이상
	2. 갱신 이상
	3. 삭제 이상
3. 데이터 무결성 유지
4. 논리적 구조 명확화

##### 정규화 단계
**제1정규형(1NF)**
- 원자성: 모든 속성이 원자값을 가져야 함
- 반복 그룹 제거

**제2정규형(2NF)**
- 부분 함수 종속 제거
- 복합키의 일부에만 종속되는 속성 분리

**제3정규형(3NF)**]
- 이행 함수 종속 제거
- 일반 속성 간의 종속성 제거

**BCNF(Boyce-Codd Normal Form)**
- 모든 결정자가 후보키인 상태
- 3NF보다 강화된 조건

**제4정규형(4NF), 제5정규형(5NF)**
- 다치 종속성, 조인 종속성 제거
- 실무에서는 3NF/BCNF까지 주로 적용

#### 비정규화
##### 정의
성능 향상을 위해 의도적으로 정규화 원칙을 위배해 중복을 허용하거나 테이블을 통합하는 기법
##### 목적
1. 조회 성능 향상 - 조인 연산 감소
2. 쿼리 단순화
3. 응답 시간 개선

##### 비정규화 기법
1. 테이블 병합
2. 칼럼 중복
3. 파생 칼럼 추가
4. 이력 테이블 분리
5. 테이블 수직/수평 분할
	1. **수직 분할**: 자주 사용되는 칼럼과 그렇지 않은 칼럼 분리
	2. **수평 분할**: 데이터를 특정 기준으로 분리(파티셔닝)
