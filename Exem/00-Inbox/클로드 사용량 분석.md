## 토큰 부족 원인

### 에이전트 팀 활용

기존에는 서브 에이전트 기반의 병렬 사용을 기반으로 했고 100달러 수준으로도 감당 가능했으나, 최큰 팀 에이전트 기능이 추가되며 하나의 팀으로 정확하고 독립적으로 빠르게 작업이 가능하게 되었으나 토큰 소모량이 상승하게 됨.

![[Pasted image 20260210110636.png]]

![[Pasted image 20260210110645.png]]

Ralph 루프, Oh my open code, Claude Code Agents Team 등등의 최근 개발 흐름을 봤을 때 토큰을 많이 소비할수록 Output이 좋다 라는 전제가 검증되어 가고 있음.

실제로 검증한 결과 토큰을 최대한 활용해 구체적이고 더 좋은 계획/설계를 세우고 이를 기반으로 작업을 진행하게 되면 훨씬 더 좋은 결과물이 나옴.

위 이미지는 개인적으로 활용중인 Deep Think 라는 스킬.

다양한 페르소나를 가진 에이전트 팀을 구성해 각 에이전트가 내가 질문한 무언가에 대해 각각의 생각으로 해답을 찾고 서로 비판 및 토론을 진행함.

메인 에이전트가 이 내용들을 보고 최종 답안을 생성.

이 스킬을 활용하면 토큰을 많이 사용하지만 답변의 품질이 아주 높음.
