## 목적
- 사용자가 필요한 컬럼의 너비를 직접 조정해 화면 정보 밀도와 가독성을 맞출 수 있게 한다.

## 핵심 동작
- 사용자는 헤더 경계의 리사이즈 핸들을 드래그해 컬럼 너비를 변경할 수 있다.
- 리사이즈 기능은 기본 활성화 상태이며 기능을 끄면 핸들이 표시되지 않는다.
- 드래그 중에는 가이드 라인이 보이고 실제 너비 변경은 드래그 종료 시점에 반영된다.
- 핸들을 클릭만 하고 드래그하지 않으면 너비는 변경되지 않는다.
- 마우스와 터치 입력 모두 리사이즈 동작을 지원한다.

## 상세 규칙
- 리사이즈 핸들은 리프 컬럼에만 표시되고 그룹 헤더에는 표시되지 않는다.
- 기본 너비가 지정된 컬럼은 기본적으로 리사이즈 비활성 상태로 시작한다.
- 기본 너비가 지정된 컬럼도 개별 허용 설정을 켜면 리사이즈할 수 있다.
- 개별 비활성화된 컬럼은 핸들이 없고 다른 컬럼 리사이즈에도 너비가 유지된다.
- 대상 컬럼을 확장하면 오른쪽의 리사이즈 가능한 컬럼이 시각적 순서대로 줄어들어 확장분을 흡수한다.
- 오른쪽 컬럼이 더 줄어들 수 없으면 대상 컬럼 확장분이 가로 오버플로우로 전환된다.
- 대상 컬럼을 축소하면 최소 너비 제한 아래로는 줄어들지 않는다.
- 축소 시 기존 가로 오버플로우가 있으면 축소분은 먼저 오버플로우를 줄이는 데 사용된다.
- 오버플로우 해소 뒤에도 축소분이 남으면 오른쪽의 첫 번째 리사이즈 가능 컬럼이 남은 폭을 가져간다.
- 오른쪽에 리사이즈 가능한 컬럼이 없으면 대상 컬럼만 축소되고 우측 빈 공간이 생길 수 있다.
- 컬럼 고정, 컬럼 순서 변경, 컬럼 표시/숨김이 켜져 있어도 현재 화면의 실제 컬럼 순서를 기준으로 동일 규칙이 적용된다.
- 제어 모드는 외부 상태로 너비를 관리하고 비제어 모드는 초기값 후 내부 상태로 관리한다.
- RTL 방향 설정 시 드래그 가이드 이동 방향도 반대로 적용된다.

## 제약 및 예외
- 최소 너비 제한이 적용되어 컬럼을 무제한 축소할 수 없다.
- 최대 너비 제한은 제공되지 않아 확장 시 가로 스크롤이 길어질 수 있다.
- 리사이즈 조작은 포인터 기반 입력 중심으로 동작한다.
- 셀 병합 기능과의 동시 사용은 지원 범위가 아니다.

## 비목표
- 콘텐츠 길이에 맞춘 자동 최적 너비 계산은 제공하지 않는다.
- 리사이즈 결과를 로컬 저장소나 서버에 자동 저장하지 않는다.
- 여러 컬럼을 동시에 묶어 일괄 리사이즈하지 않는다.
- 리사이즈 중 실시간 레이아웃 확정 반영 모드는 제공하지 않는다.

## 확인이 필요한 항목
- 오른쪽 흡수 컬럼이 없을 때 생기는 우측 빈 공간을 정책으로 허용할지 자동 보정할지 확인이 필요하다.
- 셀 병합과의 비호환을 런타임에서 강제 차단할지 문서 가이드로만 운영할지 확인이 필요하다.
- 제어 모드에서 상태값과 변경 콜백의 필수 조합을 어디까지 강제할지 확인이 필요하다.
- 사용자별 컬럼 너비 영속 저장을 기본 요구사항으로 포함할지 확인이 필요하다.
