## 토큰 부족 원인

### 1. 에이전트 팀 활용

- 기존: 서브 에이전트(단일 세션 내 병렬 처리) 방식 → $100 플랜으로 감당 가능
- 2026.02: 에이전트 팀(Agent Teams) 기능 도입 → 각 팀원이 독립 컨텍스트 윈도우 사용
- 토큰 소비가 활성 팀원 수에 비례하여 증가 (3명 기준 3~4배, plan 모드 시 약 7배)
- 결과: 작업 품질·속도는 향상되었으나, 현재 플랜 한도로는 지속 불가

### 2. 에이전틱 프로그래밍 트렌드 변화

- 최근 개발 흐름([RALPH Loop](https://github.com/snarktank/ralph), [Oh My Open Code](https://github.com/code-yeongyu/oh-my-opencode), [Claude Code Agent Teams](https://code.claude.com/docs/en/agent-teams) 등)에서 "토큰 소비량과 Output 품질이 비례한다"는 전제가 검증되는 추세 ([관련 연구](https://arxiv.org/html/2408.03314v1), [OpenAI o1 공식 발표](https://openai.com/index/learning-to-reason-with-llms/))
- 실제 업무 적용 결과, 토큰을 충분히 활용해 구체적인 계획/설계를 먼저 세운 뒤 작업을 진행하면 결과물 품질이 크게 향상됨
- 예시: 개인 활용 중인 "Deep Think" 스킬
  - 다양한 페르소나를 가진 에이전트 팀이 질문에 대해 각자 해답을 도출 → 상호 비판/토론 → 메인 에이전트가 종합하여 최종 답안 생성 ([Multiagent Debate 연구](https://composable-models.github.io/llm_debate/))
  - 토큰 소모량은 높지만, 단일 에이전트 대비 답변 품질이 현저히 높음
