## 논의 사항

### 26.01.27 - 상태 저장 관련
- 상태 저장 리스트 논의 필요 (widget pack, 새로고침 등 상태가 유지되어야 할 값들의 리스트)
- env 저장에 대한 세션 관리 필요 (같은 아이디를 사용하는 유저 두 명이 동시에 같은 env를 저장하면 덮어쓰기될 수 있음)

### 26.01.21 - Instance List Modal 및 컬러팔레트
**Instance List Modal**
- 대시보드 생성 시 Name/Note 값 할당 위치
- (우선 대시보드 생성 모달에는 Name/Note Input 요소 유지, Instance List로 접근 시 Name, Note Input 요소 숨김 처리)

**컬러팔레트**
- 팔레트(템플릿)를 통해 일괄 적용 시 팔레트 내 색상 수보다 인스턴스 수가 많을 경우 처리 방식 → 일단은 지금처럼 하나하나 적용되게
- 중복 색상 허용 여부 → 일단은 지금처럼 중복 허용되게
- 테이블 툴바 내 컬러팔레트에서의 custom 색상 값 의미

### 26.01.16 - 그룹화 및 레이아웃

**그룹화**
- 그룹과 그룹끼리 그룹핑이 가능한 경우가 있는지 → 풀어서 하나의 그룹으로
- 그룹화를 할 때 아이콘 오버레이만 뜨는데, 그룹 내부에 플레이스홀더를 표시할지 여부
- 그룹 + 위젯 끼리 비빌때는 ok
- 위젯 + 위젯으로 그룹화 할 때, 그룹의 헤더를 위해서 로우가 1만큼 더 커져야하는데, 커지면서 레이아웃이 변경될 수 있음 → 사용성 고려해서 추가 검토 필요
- 그룹 접었을 때 아래 위젯들의 재배치를 어떻게 해야할지 논의 필요 (기존 grid-layout의 vertical 기능을 쓸지, 아래 있는 위젯들의 상하 움직임을 다시 계산할지)
- 접는 기능 꼭 필요한가?

**다킹 프레임 / Layout**
- 그리드 레이아웃의 가로폭 제한 (12셀)
- 위젯 최소 사이즈 정의 필요
- 드래그 드롭시 선택된 위젯들 이동시 해당 레이아웃을 유지한 채로 이동 or compact가 적용된 채로 이동

**Tab 모드**
- Tab list가 많을 때 표현 방법

**위젯 팩**
- 생성시 타겟 인스턴스: 명시되어있지 않음, 전부? 빈배열? 혹은 선택 모달?
- 커스텀 위젯 팩 이미지: 그냥 타입에 따라야할지... > 따라야 할듯?
- 위젯 크기(w,h): 위젯 크기도 저장해야 하는지

## 위젯 빌더 관련

### 스캐터 테이블 공통
- 지표의 타입에 따라 포매터의 조건 적용 방식이 변해야 하는지? 그렇다면 어떻게 변해야 하는지?

### 스캐터 차트
- 스캐터 차트는 어떤 값을 차트로 그리는지?
- 각 칼럼들이 하나의 레전드가 되는건지?
- 스캐터 차트 칼럼 포매팅 적용시 UI는?
- 스캐터 차트 순서 변경시 무슨 일이 일어나야 하는지?

### 테이블
- 테이블 그룹바이 True로 전환시 기존 칼럼들은 어디로 사라진건지?
- 테이블에 그룹화 되었을 때의 UI, 트리 그리드의 UI 없음
