# 사고 과정 마크다운 렌더링 적용

#resume-with-ai #즉시 #UX

## 요약
챗봇 사고 과정(reasoning) 영역에서 마크다운 문법(`**bold**` 등)이 렌더링되지 않고 원시 텍스트로 노출되는 문제 수정

## 왜?
- 사고 과정 텍스트에 `**bold**` 마크다운이 스타일 적용 없이 그대로 보여 가독성이 떨어짐
- AI 어시스턴트의 추론 과정을 사용자가 이해하기 어려움
- 전문성 있는 제품 인상을 주기 위해 UI 완성도 필요

## 어떻게?
1. 사고 과정(reasoning) 표시 컴포넌트 확인 — `@assistant-ui/react` 커스텀 렌더러 또는 직접 구현된 부분 탐색
2. 해당 영역에 마크다운 렌더링 적용 — `react-markdown` 등 라이브러리 활용 또는 기존 마크다운 렌더러 재사용
3. 사고 과정 특성에 맞게 스타일링 — 본문 답변과 구분되도록 시각적 차별화 유지

## 예상 효과
- 사고 과정의 가독성 향상 (bold, 리스트 등 시각적 구조화)
- 챗봇 UI 완성도 개선
