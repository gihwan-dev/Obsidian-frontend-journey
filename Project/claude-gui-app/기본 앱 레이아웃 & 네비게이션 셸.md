# 기본 앱 레이아웃 & 네비게이션 셸

#claude-gui-app #즉시 #UI

## 요약
앱의 기본 레이아웃 구조(사이드바, 메인 영역, 상태바)와 페이지 간 네비게이션을 구성한다.

## 왜?
- 모든 기능 컴포넌트가 배치될 공간이 필요
- 초기에 레이아웃을 확정해야 이후 기능별 개발 시 충돌 없이 독립적으로 진행 가능
- 사이드바 기반 네비게이션은 다중 세션, 워크플로우 등 복잡한 기능을 수용할 수 있는 구조

## 어떻게?
1. 레이아웃 구조 설계:
   - 좌측 사이드바: 세션 목록, 프로젝트 탐색기, 설정 진입점
   - 중앙 메인 영역: 터미널, 워크플로우 에디터, 모니터링 등 주요 콘텐츠
   - 하단 상태바: 현재 세션 정보, 토큰 사용량, 연결 상태
2. React Router 또는 자체 라우팅으로 페이지 전환
3. 사이드바 ↔ 메인 영역 크기 조절 (resizable pane)
4. Tailwind CSS 또는 CSS Module로 스타일링 기반 구축

## 예상 효과
- 이후 모든 기능 개발이 일관된 레이아웃 안에서 진행
- 사용자가 앱의 전체 구조를 직관적으로 파악 가능
- 다중 패널 구성으로 동시 정보 확인 가능
