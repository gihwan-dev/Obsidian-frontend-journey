# 세션 간 컨텍스트 전달

#claude-gui-app #중기 #워크플로우 #컨텍스트

## 요약
워크플로우 내에서 이전 세션의 결과를 자동 compact하여 다음 세션의 프롬프트에 주입하는 컨텍스트 전달 메커니즘을 구현한다.

## 왜?
- 워크플로우의 세션들은 서로 독립적이지만, 이전 작업의 맥락을 알아야 의미 있는 후속 작업 가능
- "로그인 페이지 만들기" → "방금 만든 코드에 테스트 작성" 같은 연쇄 작업에서 컨텍스트 연속성 필수
- 수동으로 요약을 작성하여 전달하면 자동화의 의미가 없어짐

## 어떻게?
1. 세션 완료 시 자동 compact 실행:
   - Claude Code의 `/compact` 기능 활용 또는 별도 요약 프롬프트 실행
   - compact 결과(요약본)를 JSON으로 저장
2. 다음 노드 시작 시 프롬프트 주입:
   ```
   "이전 작업 요약: {compact_result}
    이번 작업: {session_prompt}"
   ```
3. 조건 노드에서도 compact 결과 파싱하여 분기 판단에 활용
4. compact 결과 포맷 표준화 (구조화된 섹션: 변경사항, 결과, 미해결 이슈 등)

## 예상 효과
- 워크플로우 세션 간 자연스러운 문맥 이어가기
- 자동화 품질 향상 (후속 세션이 이전 맥락을 이해)
- 조건 분기의 정확도 향상
