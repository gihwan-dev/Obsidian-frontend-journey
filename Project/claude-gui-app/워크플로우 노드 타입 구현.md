# 워크플로우 노드 타입 구현

#claude-gui-app #중기 #워크플로우

## 요약
워크플로우 시스템의 6가지 노드 타입(세션, 조건, 루프, 트리거, Git, 알림)을 구현한다.

## 왜?
- 노드 에디터의 껍데기만으로는 워크플로우를 실행할 수 없음
- 각 노드 타입이 고유한 역할과 설정을 가져야 다양한 자동화 시나리오 구성 가능
- 스펙에 정의된 6가지 노드 타입은 일반적인 개발 워크플로우를 충분히 커버

## 어떻게?
1. **세션 노드**: Claude Code 세션 실행. 프롬프트, 모델, 스킬, MCP 설정 포함
2. **조건 노드**: 이전 노드 출력 기반 분기. 조건식 (exit code, 텍스트 매칭 등)
3. **루프 노드**: 무한 루프 모드 컨테이너. 종료 조건 포함
4. **트리거 노드**: 워크플로우 시작 조건. 수동, 스케줄, 파일 변경 감지
5. **Git 노드**: 커밋, 체크포인트, PR 생성 등 git 작업
6. **알림 노드**: 사용자 확인 요청, 중간 리뷰 포인트, 시스템 알림
7. 각 노드 타입별 설정 패널 UI 구현
8. 노드 실행 결과 → 다음 노드 입력으로 전달하는 데이터 플로우

## 예상 효과
- 다양한 자동화 시나리오 구성 가능
- 기능 구현 → 테스트 → 조건부 수정 → PR 같은 복합 워크플로우 자동화
- 확장성: 향후 새로운 노드 타입 추가 용이
